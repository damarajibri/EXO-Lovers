📋 Langkah-Langkah Migrasi (Urutan Eksekusi)

Pre-Migration: Inventory & Validasi
• Dokumentasikan semua IP mapping (lama → baru) untuk 30+ source IPs dan 15+ dest IPs
• Validasi semua port (57183, 179, 7777, 7077, 12003, 12004, 10031, 10022, 80, ICMP) sudah dibuka di firewall DC baru
• Siapkan rule Palo Alto di DC baru yang mirror dari DC eksisting
• Koordinasi dengan semua partner (Mitrakom, Duitku, Shopee, PLN, Finnet, SYB, Pertamina, Nicepay, iSaku)
Fase 1: Migrasi FIBERSTAR (Risiko RENDAH)
• Hanya ICMP monitoring — 172.28.145.13 → 10.219.3.186 & 10.219.3.190
• Pindahkan monitoring ping, validasi reachability
• Rollback mudah jika gagal — tidak ada traffic aplikasi
Fase 2: Migrasi ICON (Risiko SEDANG)
• Pindahkan monitoring ping (172.19.91.31, 172.18.65.229, 172.28.145.13)
• Pindahkan BRI-PERTAMINA: 172.28.94.3/4 → 172.28.108.49 port 80
• Pindahkan BRI-PERTAMINA-PATRANIAGA: 172.18.104.137 → 172.28.108.77 port 80
• Catatan: banyak session "incomplete" — validasi koneksi end-to-end
Fase 3: Migrasi INDOSAT (Risiko SEDANG)
• Pindahkan monitoring ping (3 flow)
• Pindahkan BRI-DUITKU via Indosat: 172.28.94.3/4 + 172.28.73.254 → 172.28.108.47 port 7777
• Koordinasi dengan tim Duitku — traffic juga ada di backhaul Lintas
